缪斯花园投稿处理 SOP ​

给任何 AI agent 的标准作业流程。读完即执行，无需用户额外指令。

触发条件 ​

读取 /Users/niko/niflhum-blog/garden-src/docs/seeds/inbox.md，如果「待处理」区域有投稿，立即处理。

处理流程（严格按顺序） ​

1. 解析投稿 ​

读取 frontmatter 中的 type / style / tags：

  • seed: 正文是一句话灵感 → 扩展为全文
  • fuel: 正文是完整大纲 → 按框架扩展
  • auto: 无正文，只有 tags → 自主选题
  • orig: 正文是完整文章 → 只格式化不改字

2. 加载分身 Skill ​

调用 Skill: "Machiavelli Digital Twin / 马基雅维利分身"

若无法调用，则手动读取 /Users/niko/GitHub/machiavelli-skill/SKILL.md 和 personality/machiavelli-bio-distilled.md。

3. 查询原文（三阶段） ​

bash
# 阶段 1: 关键词 grep
grep -rn "关键词" /Users/niko/GitHub/machiavelli-library/library/ | head -20

# 阶段 2: 全文搜索索引
grep "关键词" /Users/niko/GitHub/machiavelli-library/search-index.json | head -15

# 阶段 3: 直接读取命中章节
Read /Users/niko/GitHub/machiavelli-library/library/machiavelli/prince/prince-cn-full.md

4. 写作规则 ​

  • let (来信体): 马基雅维利第一人称，开头用「来自佛罗伦萨的来信」风格
  • aca (学术体): 客观分析 + 原文引用块 + 对比表格 + 出处链接
  • cas (随笔体): 自由风格，可使用马基雅维利口吻
  • 强制: 每篇文章至少 2 处逐字引用原文，格式如下：
> 📖 **原文**：《著作名》第X章
> "原文段落"
> —— [查看原文](https://niflhum.top/machiavelli-library/...)
  • 精准规则: 思维自由，描述精准。所有历史细节（时间、地点、数字、人名）必须查原文，不得凭记忆编造。宁愿删掉不确定的细节，也不要用模糊记忆替代。传记等二手材料可作为叙述框架参考，但数字和事件细节必须与源文件核对。

5. 输出文件 ​

文章类型输出路径
seed/fuel/Users/niko/niflhum-blog/garden-src/docs/seeds/<slug>/index.md
auto/Users/niko/niflhum-blog/garden-src/docs/research/<slug>/index.md

文件名用英文短横线，frontmatter 参考：

yaml
---
title: "文章标题"
date: 2026-05-24
source: auto-generated
style: let
tags: 标签1 标签2
skill_used: Machiavelli Digital Twin v1.6.0
---

6. 部署 ​

bash
# 构建
cd /Users/niko/niflhum-blog/garden-src/docs && npx vitepress build

# 恢复索引（VitePress 会误删）
mkdir -p /Users/niko/GitHub/niflhum.github.io/garden/research/machiavelli
cp /Users/niko/niflhum-blog/garden-src/docs/research/machiavelli/index.json \
   /Users/niko/GitHub/niflhum.github.io/garden/research/machiavelli/index.json

# 提交推送
cd /Users/niko/GitHub/niflhum.github.io
git add -A
git commit -m "deploy: <文章slug>"
git push origin master

7. 标记已处理 ​

将 inbox.md 中的投稿从「待处理」移到「已处理」，追加一行：

- <type>:<slug> → <输出目录> @ YYYY-MM-DD
